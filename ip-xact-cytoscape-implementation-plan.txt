PROJECT: ip-xact (Cytoscape.js variant)
Hardware Architecture Explorer — Implementation Plan for AI Coding Agent
=========================================================================

CHANGES FROM ORIGINAL PLAN (applied per user request)
-------------------------------------------------------
1. REMOVE node expand/collapse entirely. No expanded state, no
   expandedLabel, no .expanded CSS class, no single-expand-at-a-time
   rule, no expandedNodeId in selectionStore. Nodes are always in their
   collapsed (default) card form. The Inspector panel (Section 12) is the
   sole place where port/register detail is displayed.

2. KEEP zoom/pan exactly as planned (native Cytoscape, toolbar buttons).

3. LARGE INPUT HANDLING: the graph must stay interactive with hundreds to
   thousands of nodes. Apply all performance mitigations from Section 14
   from the start, not as a later pass:
   - Enable hideEdgesOnViewport: true and hideLabelsOnViewport: true on
     the cy core options at construction time.
   - Use cy.batch() for every multi-element update.
   - Run ELK layout in a Web Worker via the preset-layout escape hatch
     (see Section 14) as the DEFAULT path, not the fallback, so the main
     thread never blocks on layout regardless of graph size.
   - Prefer stylesheet class toggles over per-element .style() calls.

4. GRAPHICAL LIVE MAPPING: the rendered diagram must look like a real
   EDA / SoC circuit schematic, not a generic node-link graph. Concretely:
   - Use 'curve-style': 'taxi' for ALL edges (right-angle orthogonal
     routing), which is the standard look for hardware block diagrams.
   - Use 'edge-distances': 'node-position' and explicit
     'source-endpoint'/'target-endpoint' anchoring so wires connect at
     the correct cardinal face of each block (outputs leave from the
     right/bottom, inputs arrive at the left/top, matching ELK port
     assignment).
   - Add a dot-grid CSS background pattern on the canvas container div
     (pure CSS, not a Cytoscape feature) to reinforce the PCB/schematic
     aesthetic.
   - Node shape must be 'rectangle' (not round-rectangle) — hard-edged
     blocks are standard in hardware diagrams.
   - Each node must display a small type-icon glyph inside the block
     using Cytoscape's background-image (inline SVG data URI) alongside
     the component name label.
   - Edge labels: show the port pair (sourcePortId → targetPortId) as a
     small label on each edge using Cytoscape's 'label' on edges, with
     font-size 9px and a white/dark background pill so labels don't bleed
     into the diagram. Use 'text-rotation': 'autorotate' so labels align
     with the wire direction.
   - Use ELK's 'layered' algorithm with 'elk.direction': 'DOWN' so the
     hierarchy flows top-to-bottom (clock/reset at top, CPU next, bus in
     the middle, peripherals at the bottom) matching standard SoC block
     diagrams.


NOTE ON THIS VARIANT
----------------------
This replaces the React Flow + standalone ELK.js plan with Cytoscape.js
as the graph engine, per these confirmed decisions:
  - Nodes use NATIVE Cytoscape canvas styling (no HTML/DOM overlay on
    the canvas). Node expand/collapse has been REMOVED — nodes are
    always in their default card form; all detail is in the Inspector.
  - Layout uses the cytoscape-elk extension, so ELK's hierarchical
    "layered" algorithm is preserved. ELK runs in a Web Worker by
    default (see Section 14).
  - The app shell stays React + TypeScript + Zustand + TailwindCSS.
    react-cytoscapejs is the bridge component between React and the
    Cytoscape core instance.
  - The DOMAIN input JSON (components/types/ports/registers/connections)
    is UNCHANGED from the original plan. A dedicated adapter converts it
    into Cytoscape's own element JSON format (the format Cytoscape
    actually ingests) — see Section 5 and Section 7. Keep these two JSON
    shapes separate; do not hand-author Cytoscape's format directly as
    your source data.


1. PROJECT OVERVIEW
---------------------
Build a modern Hardware Architecture Explorer web app named "ip-xact".

Stack:
- React + TypeScript (strict mode)
- Cytoscape.js (core graph rendering engine, canvas-based)
- react-cytoscapejs (React wrapper/bridge for the Cytoscape instance)
- cytoscape-elk (extension wrapping elkjs, gives hierarchical layout)
- elkjs (peer dependency; also used directly in Web Worker for off-thread
  layout — import elkjs/web in the worker)
- TailwindCSS (styling for UI chrome: panels, search bar, toolbar, badges
  — NOT for node visuals, since nodes are canvas-drawn via Cytoscape's
  own stylesheet, not DOM/CSS)
- Zustand (state management, split into multiple slices/stores)
- Fuse.js (fuzzy search)
- Vite (build tool)

Input: same JSON architecture model as before — components, component
types, ports, registers, directed connections.

Output: an interactive, zoomable, searchable Cytoscape diagram of the
architecture rendered as a proper SoC/EDA block diagram (orthogonal
taxi-routed wires, hard-edged rectangular blocks, dot-grid background,
port-pair edge labels), dark-themed, supporting graphs with hundreds to
thousands of nodes without blocking the UI.

4. TYPE DEFINITIONS (Phase 1 — build first)
-----------------------------------------------
types/architecture.ts — UNCHANGED from the original plan:
- ComponentType: "cpu" | "bus" | "memory" | "peripheral" | "interface" |
  "clockReset" | "custom"
- Port: { id, name, direction, width? }
- Register: { id, name, address?, description? }
- Component: { id, name, type: ComponentType, ports: Port[],
  registers: Register[] }
- Connection: { id, sourceComponentId, sourcePortId, targetComponentId,
  targetPortId }
- ArchitectureModel: { components: Component[], connections: Connection[] }

types/cytoscapeElements.ts — defines the shape Cytoscape requires:
- CytoscapeNodeData: { id: string, label: string, type: ComponentType }
  NOTE: isExpanded and expandedLabel fields are REMOVED.
  Element shape: { group: 'nodes', data: CytoscapeNodeData }
- CytoscapeEdgeData: { id: string, source: string, target: string,
  sourcePortId: string, targetPortId: string, label: string }
  NOTE: label = `${sourcePortId} → ${targetPortId}`, precomputed in
  cytoscapeAdapter. Element shape: { group: 'edges', data: {...} }
- Keep this file separate from architecture.ts — domain model vs.
  rendering-library shape.

lib/transform/colorMap.ts — same color tokens as before, defined once
and consumed two ways:
  cpu          -> blue   (#388bfd base, #1c3d5e fill)
  bus          -> purple (#bc8cff base, #2d1f52 fill)
  memory       -> green  (#3fb950 base, #1a3a23 fill)
  peripheral   -> orange (#f78166 base, #3d2218 fill)
  interface    -> cyan   (#39d353 base, #183d2d fill)
  clockReset   -> red    (#ff7b72 base, #3d1a1a fill)
  custom       -> gray   (#a371f7 base, #2d1f52 fill)
Export both: (a) hex/Tailwind tokens for Inspector badges, and (b) a
function `buildCytoscapeColorStyles(colorMap)` that generates the
Cytoscape stylesheet selector rules for each type (Section 9). Use
`satisfies Record<ComponentType, NodeColorTokens>` so adding a new type
without updating the map is a compile error.

lib/transform/nodeIcon.ts — NEW. Returns a small inline SVG data URI
(roughly 24×24) per ComponentType, used as Cytoscape node
background-image. Keep icons simple single-path shapes:
  cpu       -> simple chip/square with small squares at edges
  bus       -> horizontal lines (bus traces)
  memory    -> stacked rectangles
  peripheral -> gear/cog outline
  interface -> bidirectional arrow
  clockReset -> clock face (circle + hands)
  custom    -> question mark or generic block
Export: `getNodeIconDataUri(type: ComponentType): string`

6. ZUSTAND STORE DESIGN (Phase 2)
---------------------------------------
Same three-store split as the original plan, with these changes:

a) architectureStore.ts — unchanged: holds ArchitectureModel plus
   precomputed Map<componentId, Component>, Map<componentId,
   Connection[]> incoming, Map<componentId, Connection[]> outgoing,
   built once at load.

b) graphStore.ts — holds Cytoscape ELEMENT DATA (the output of
   cytoscapeAdapter.ts) and the current layout name. Does NOT hold the
   live `cy` core instance.

c) selectionStore.ts — shape: selectedNodeId, highlightedNodeIds (Set),
   dimmedNodeIds (Set), highlightedEdgeIds (Set).
   expandedNodeId is REMOVED. There is no expandNode action.
   Single-select drives highlighting only (Section 11).

lib/cytoscape/instanceManager.ts — singleton that holds the actual `cy`
instance once CytoscapeCanvas mounts. Exposes:
  - selectNode(id: string | null)
  - highlightConnections(id: string)
  - focusNode(id: string)
  - resetView()
  expandNode is REMOVED from this interface.

Rule (unchanged): components subscribe via narrow Zustand selectors,
never the whole store object.


7. JSON INGESTION — DOMAIN MODEL TO CYTOSCAPE ELEMENTS (Phase 2/3)
-----------------------------------------------------------------------
lib/cytoscape/cytoscapeAdapter.ts — pure function:
  (model: ArchitectureModel) => { nodes: CytoscapeNodeElement[],
                                    edges: CytoscapeEdgeElement[] }
- For each Component, emit a node element:
    { group: 'nodes', data: { id, label: name, type } }
  (no isExpanded, no expandedLabel)
- For each Connection, emit an edge element:
    { group: 'edges', data: {
        id,
        source: sourceComponentId,
        target: targetComponentId,
        sourcePortId,
        targetPortId,
        label: `${sourcePortId} → ${targetPortId}`
      }
    }
- Run once when the model loads; do not re-run on every render.
- The resulting elements array is passed to <CytoscapeComponent
  elements={...} stylesheet={...} /> on initial mount only.


8. ELK LAYOUT — WEB WORKER PATH (Phase 3) [DEFAULT, not escape hatch]
-----------------------------------------------------------------------
lib/cytoscape/elkWorker.ts — a Vite web worker module:
  - Receives a message of type { model: ArchitectureModel, options: ElkLayoutOptions }
  - Imports ELK directly: import ELK from 'elkjs/wasm/elk.bundled.js'
    (or 'elkjs/web' depending on bundler resolution — test both)
  - Builds an ELK graph from the model (same shape as elkLayoutConfig.ts)
  - Calls elk.layout(graph) and posts back:
      { positions: Record<string, { x: number, y: number }> }
  - Worker never touches the DOM or cy instance.

hooks/useElkLayout.ts:
  - Creates the worker once on mount: new Worker(new URL('../lib/cytoscape/elkWorker.ts', import.meta.url), { type: 'module' })
  - Sends the model to the worker after initial cy mount.
  - On message received, applies positions imperatively:
      cy.batch(() => {
        cy.nodes().forEach(node => {
          const pos = positions[node.id()];
          if (pos) node.position(pos);
        });
      });
      cy.layout({ name: 'preset' }).run();
      cy.fit(undefined, 60);
  - Shows a loading indicator in GraphToolbar while layout is running.

lib/cytoscape/elkLayoutConfig.ts — exports the ELK options object used
by the worker:
  {
    id: 'root',
    layoutOptions: {
      'elk.algorithm': 'layered',
      'elk.direction': 'DOWN',
      'elk.layered.spacing.nodeNodeBetweenLayers': '80',
      'elk.spacing.nodeNode': '40',
      'elk.layered.nodePlacement.strategy': 'NETWORK_SIMPLEX',
      'elk.edgeRouting': 'ORTHOGONAL',
      'elk.layered.unnecessaryBendpoints': 'true',
    }
  }

10. NODE DESIGN (Phase 4)
--------------------------
Defined entirely in lib/cytoscape/styleSheet.ts. Expand/collapse is
REMOVED. All nodes are always in their default card form.

- Shape: 'shape': 'rectangle' (hard-edged, not round-rectangle —
  matches EDA/hardware diagram conventions).
- Default node size: width 160, height 70. These are fixed; no resize
  on interaction.
- Color-coding by type via selectors:
    { selector: 'node[type = "cpu"]',
      style: { 'background-color': colorMap.cpu.fill,
               'border-color': colorMap.cpu.base } }
  generated programmatically from colorMap.ts for all seven types.
- Top accent bar: simulate with a compound node child OR with a
  'border-top-color' + 'border-top-width' trick. Simplest: use a
  2px top border in the type accent color, 0.5px on the other three
  sides. Cytoscape supports 'border-top-width' and 'border-top-color'
  from v3.x — use if available, otherwise use a uniform border.
- Icon: set 'background-image' to the data URI from nodeIcon.ts,
  'background-width': 20, 'background-height': 20,
  'background-position-x': 8, 'background-position-y': 25 (bottom-left
  corner of the node, leaving top area for the label).
- Label: 'label': 'data(label)', 'text-valign': 'top',
  'text-halign': 'center', 'text-margin-y': 8, font-size 12px,
  font-weight 500.
- Hover feedback: .hovered class — thicker border + brighter border color:
    'border-width': 2, 'border-color': <type accent>
  with transitions:
    'transition-property': 'border-width, border-color',
    'transition-duration': '150ms'
- Selection: :selected pseudo-class — bright accent border, no fill
  change needed (the Inspector makes the selection clear in the DOM).
- NO .expanded class, NO expandedLabel, NO size transitions for
  expand/collapse. Those are removed.


11. CONNECTION / EDGE DESIGN (Phase 4+5)
-----------------------------------------
- curve-style: 'taxi' for ALL edges — right-angle orthogonal routing,
  the standard for hardware block diagrams.
- taxi-direction: 'downward' to bias vertical-first routing consistent
  with the DOWN ELK direction.
- target-arrow-shape: 'triangle-backcurve' (or 'triangle') — clear
  directionality.
- Edge base style:
    'line-color': '#30363d'
    'target-arrow-color': '#30363d'
    'width': 1
    'opacity': 0.7
- Edge label:
    'label': 'data(label)'   (the "srcPort → tgtPort" string)
    'font-size': 9
    'color': '#8b949e'
    'text-background-color': '#0d1117'
    'text-background-opacity': 0.85
    'text-background-padding': '2px'
    'text-rotation': 'autorotate'
    'text-margin-y': -6
  If edge labels are too noisy on large graphs, add a toggle in
  GraphToolbar to show/hide them via a stylesheet class swap.
- Highlight state (on node select, inside cy.batch()):
    1. cy.elements().removeClass('highlighted dimmed')
    2. selected node + connectedEdges() + their connectedNodes()
       get addClass('highlighted')
    3. everything else gets addClass('dimmed')
  Stylesheet rules:
    '.highlighted node': { opacity: 1, 'z-index': 999 }
    '.highlighted edge': { opacity: 1, 'line-color': '#f0883e',
                            'target-arrow-color': '#f0883e', width: 2 }
    '.dimmed': { opacity: 0.1 }
- Precomputed adjacency from architectureStore drives highlighting —
  never filter the full edge array at selection time.


12. INSPECTOR PANEL (Phase 6)
------------------------------------
Pure React/Tailwind DOM panel, reads from architectureStore's precomputed
incoming/outgoing maps. This is the ONLY place port and register detail
is displayed (since node expand is removed).

Panel sections:
  - Component name (large) + type badge
  - Component ID (small, monospace)
  - Ports list: each port shows name, direction badge (IN/OUT/INOUT),
    and optional bit-width
  - Registers list: name + optional address + optional description tooltip
  - Drives → list: connected output components, each a clickable chip
    that calls focusNode(id)
  - ← Receives from list: connected input components, same chip pattern

Clicking a connection chip calls the shared focusNode(id) from
instanceManager — identical to search result selection behavior.


13. SEARCH (Phase 7)
----------------------------
- Fuse.js index built once at load over component name/type/id.
- SearchBar.tsx: debounced input (~150ms).
- On result selection, call one shared focusNode(id) function (in
  instanceManager.ts) used identically by search, direct node clicks,
  and Inspector cross-links:
    cy.animate({ fit: { eles: cy.getElementById(id), padding: 80 },
                  duration: 400 });
    selectionStore.selectNode(id);
  All three interaction paths must funnel through this one function.


14. PERFORMANCE REQUIREMENTS (apply from Phase 1, not as a later pass)
-----------------------------------------------------------------------
MANDATORY FROM THE START (not a polish pass):

- cy core options at construction: hideEdgesOnViewport: true,
  hideLabelsOnViewport: true, textureOnViewport: true.
- cy.batch() wraps EVERY multi-element update (highlight, dim, reset).
- Stylesheet class toggles only — no per-element .style() calls when
  touching many elements.
- ELK runs in a Web Worker by DEFAULT (see Section 8). The main thread
  never runs ELK directly.
- Do not re-run layout on any interaction — layout runs once on load
  (and optionally on explicit "Re-layout" toolbar button press).
- React re-renders must stay confined to Inspector/Search/UI chrome.
  The CytoscapeCanvas React wrapper should not re-render after initial
  mount. All graph updates go through instanceManager → cy imperative
  API, bypassing React.
- For very large graphs (200+ nodes): add a "Loading layout…" spinner
  while the worker runs, and consider a progressive render (show nodes
  immediately with random positions, then animate to ELK positions when
  ready).
- Large file upload: when user loads a JSON file at runtime, parse it
  in a try/catch, validate the shape (components array + connections
  array both present), then feed through cytoscapeAdapter and rebuild
  the cy instance. Do not add all elements via cy.add() in a loop
  outside cy.batch().


15. VISUAL QUALITY / THEME (apply throughout)
----------------------------------------------------
Target aesthetic: GitHub dark theme meets KiCad/Quartus block diagram.

- Dark background: #0d1117 with dot-grid pattern (Section 9).
- Node blocks: hard-edged rectangles, dark type-colored fills, bright
  type-colored borders and accent top bar, white component name label.
- Edges: orthogonal taxi-routed wires in dim gray (#30363d), brightening
  to orange (#f0883e) on highlight.
- Port-pair labels on edges: 9px, muted, with a near-black background
  pill so they read over the grid.
- UI chrome (toolbar, inspector, search): Tailwind dark classes matching
  GitHub's dark palette — bg-[#161b22], border-[#30363d], text colors
  from the gray-300/400/500 family.
- colorMap.ts is the single source of truth — Inspector badges and graph
  node colors must never drift apart.
- Transition timing: hover border 150ms. No other transitions (expand
  transitions removed).
